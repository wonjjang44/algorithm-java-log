## 요약
정수 배열 numLog가 주어질 때,
umLog[0]에서 부터 시작해 "w", "a", "s", "d"로 이루어진 문자열을 입력으로 받아 순서대로 다음과 같은 조작을 했다고 가정

- "w" : 수에 1을 더한다.
- "s" : 수에 1을 뺀다.
- "d" : 수에 10을 더한다.
- "a" : 수에 10을 뺀다.

그리고 매번 조작을 할 때마다 결괏값을 기록한 정수 배열이 numLog, 
즉 numLog[i]는 numLog[0]로부터 총 i번의 조작을 가한 결과가 저장되어 있다.
- --

## 접근
- numLog 0번 째 인덱스는 고정된 값 0을 가진다
- numLog[i + 1] = numLog[i] + x
- numLog[i + 1] - numLog[i] = numLog[i] - numLog[i] + x
- x = nunLog[i + 1] - numLog[i]
- -- 

## 회고
- 인접한 인덱스에 해당하는 값을 버블 정렬처럼 이중 for문을 돌면서<br/>
  하나씩 비교(값의 차이를 비교)하여 찾으려고 했으나 막힘
- 문제를 잘 읽어보니 매번 조작을 할 때마다 결괏값을 기록한 정수 배열이 numLog이며<br/>
  numLog[i]는 numLog[0]로부터 총 i번의 조작을 가한 결과가 저장된다 라고 함
- 그렇다면 0번 인덱스 기준으로 1번 인덱스 값은 0번 인덱스 기준 1번 입력된 값이라는 뜻
- 최초 로그가 [0, 10] 이라고 가정
  - => 입력한 값 = 다음 값 - 현재 값
  - 입력한 값 => numLog[i + 1] - numLog[i] => 10 - 0 = 10 ==> d
  
- for문을 numLog.length - 1한 이유는 로그가 12개라면 11번 입력된 것이므로 -1
