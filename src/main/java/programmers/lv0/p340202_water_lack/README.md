# p340202 - 물 부족

- 링크: https://school.programmers.co.kr/learn/courses/30/lessons/340202
- --

## 요약
이번달부터의 월별 물 사용량 변화를 예측한 값을 리스트에 담는다
> - [10, -10, 10, -10, 10, -10, 10, -10, 10, -10]<br/>
> - 리스트의 각 원소는 해당 월의 물 사용량이 전 달에 비해 몇 % 만큼 증가 또는 감소하는지를 나타낸다<br/>
> - 예를 들어, 이번달의 물 사용량 (리스트의 첫 번째 원소)은 지난 달 보다 10% 증가한 값이며,<br/>
>   다음 달(리스트의 두 번째 원소)의 물 사용량은 이번달 사용량에서 10%만큼 감소한 값.
  
현재 저수지에 저장된 물의 양을 나타내는 정수 storage <br/>
지난 달 물 사용량을 나타내는 정수 usage,<br/>
월별 물 사용량이 전 달 대비 어떻게 변하는지 저장된 정수 리스트 change<br/>
변수가 주어질 때, 몇 달 뒤 물이 부족해지는지 return하는 메서드 작성
- --

## 접근
- 월별 물 사용량의 변화를 저장한 배열인 change 길이 만큼 반복하면서 월별 물 사용량의 change[i]%한 결과와<br/>
  총 물 사용량의 대한 누적합계를 계산한다
- 최종적으로 출력된 총 물 사용량과 파라미터로 전달받은 저수지에 저장된 물의 양을 비교하여 이하면 -1, 초과하면<br/>
  i를 리턴(몇 번째 달에서 부족해지는지 알기 위해) 
- -- 

## 회고
- 월 별 물 사용량 계산 시 usage + (usage * (percent / 100)) 이런식으로 로직을 작성했었다.
- 지속적으로 percent / 100 값이 0이 출력되고 있었음
  - 자료형이 int형이므로 소수점이 버려진다는 사실을 잊고 있었음.
  - 괄호를 없애고 sage * percent / 100로 변경함
