## 요약
정수 배열 2개가 주어질 때, 아래와 같은 작업을 반복한다.<br/>
> - 짝수 인덱스에서는 arr에서 query[i]번 인덱스를 제외하고 배열의 query[i]번 인덱스 뒷부분을 잘라서 버린다.
> - 홀수 인덱스에서는 arr에서 query[i]번 인덱스는 제외하고 배열의 query[i]번 인덱스 앞부분을 잘라서 버린다.
 
위 작업을 마친 뒤 남은 정수 배열의 부분 배열을 리턴하는 메서드 구현. 
- --

## 접근
- 두 번째로 받은 정수 배열의 값 기준으로 배열을 잘라야 함
- 해당 값들을 짝수/홀수 검사하여 앞/뒤로 배열을 자른다.
- --

## 회고
- 기존 배열을 조건에 맞게 자르려고 함.
  - Arrays.copyOfRange 메서드를 사용하여 구현했지만 테스트 케이스에 전부 통과하지 못함.
- 짝수/홀수를 판별하는 조건문에 queries[i]라고 기입해 둠.
  - 문제를 잘못 이해함. => 두 번째 정수 배열로 넘어온 값 중 [4, 1, 2] 값을 기준으로 짝수/홀수를 판별함.
  - 이 값이 아닌 인덱스 값(0, 1, 2)로 판별하는 것이였음.. 따라서 i % 2 == 0가 맞음.


- 다른 방식으로 짝수/홀수 일 때, 배열 자체를 변경하는 것이 아닌, 인덱스 값만 계산하여
  루프가 끝난 시점에 최종적으로 계산된 인덱스 번호를 사용하여 배열을 복사하는 방식도 가능할 것 같음
  - 원본 배열이 [0, 1, 2, 3, 4, 5] 라고 가정했을 때, 현재 배열[1, 2, 3, 4]일 때,
    현재 배열 idx 0 => 원본 배열 idx = start
    현재 배열 idx 1 => 원본 idx = start + 1
    현재 배열 idx n => 원본 idx = start + n
  - 원본 idx = 현재 idx + 1
