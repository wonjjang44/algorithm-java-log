# p181894 - 2의 영역

- 링크: https://school.programmers.co.kr/learn/courses/30/lessons/181894
- --

## 요약
정수 배열이 주어질 때, 배열 안의 2가 포함된 가장 작은 연속된 부분 배열을 리턴하는 메서드 구현.<br/>
단, 2가 없는 경우 [-1] 리턴.
- --

## 접근
- 2가 처음 등장하는 첫 번째 인덱스 값과 마지막 2의 인덱스 값을 알아야 함
- 마지막 2의 인덱스 값은 lastIndexOf() 메서드를 사용하면 될 듯
- indexOf와 lastIndexOf 메서드를 사용하기 위해 정수 배열을 문자열 배열로 변경한다.
- 이후 join 메서드를 사용하여 문자열로 변경한 후 indexOf, lastIndexOf 메서드를 
  사용하여 2의 인덱스 값을 추출한다.
- copyOfRange 메서드를 사용하여 배열을 추출한다.
- --

## 회고
- 테스트 케이스에서 걸림.
- 4번째 테스트 케이스에서 정수 배열의 원소 중 10이 포함돼 있었는데 join 후 lastIndexOf로 뽑아 낼 때, 아래와 같이 출력됨
  - 1 2 1 2 1 10 2 1 => 정수 배열의 원소들
  - 121211021 => 정수 배열의 원소들을 join한 결과
  - lastIndexOf 시 두 자리 수(10)는 각 개별적으로 인식하여 2의 인덱스 위치가 틀어짐.
    - 6이 나와야하는데 7이 나옴
  - 이에 대한 보완 로직(0이 검색되면 cnt를 1씩 증가시키고 이를 lastIndexOf 값에서 빼주는 로직)을 작성했지만
    컴파일 오류가 터지고 있음. 

- 다시 한 번 더 노트에 필기하면서 돌려보니 만약 배열이 10, 2, 3, ... 이런식으로 시작돼도 바로 깨질 수 있음.
- 정수 배열을 굳이 문자열 배열로 변경할 이유가 없음.
