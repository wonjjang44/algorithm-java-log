## 요약
정해져 있는 예산으로 최대한 많은 부서의 물품을 구매할 수 있어야 한다.<br/>
각 부서가 신청한 금액만큼 모두 지원해 줘야 함.<br/>
최대 몇 개의 부서에 물품을 지원해 줄 수 있는지 return 하는 메서드 구현.<br/>

ex) 부서별 신청 금액 : [1, 3, 2, 5, 4] / 총 예산 : 9 <br/>
1, 3, 2, 5, 4 값들의 조합으로 9를 만들어낼 수 있는 최대한의 수를 뽑아낸다.<br/>
(1) 1 + 3 + 2 + 5 + 4 = 15 => 9와 일치하지 않으므로 탈락<br/>
(2) 1 + 3 + 2 + 4 = 10<br/>
(3) 1 + 3 + 4 = 8<br/>
(4) 1 + 3 + 5 = 9 => 일치 <br/>
(5) 5 + 4 => 9 > 총 예산인 9와 일치하지만 3 + 2 + 4도 9와 일치하므로 5 + 4는 사용 불가 <br/> 
=> 정해진 총 예산으로 최대로 많은 부서를 지원해 줘야 하니까<br/>
총 3개의 부서까지 지원해줄 수 있음.
- --

## 접근
- 파라미터로 받은 각 부서별 신청 금액 배열의 합이 두 번째로 받은 예산 값을 초과하면 안 된다.
- 배열의 원소의 합이 예산값과 일치하는, 즉 최대한 많은 부서를 예산과 맞게 지원해 줘야함.
- 오름차순 정렬 후 반복하며 배열의 각 원소의 합을 변수에 누적한다.
- 해당 값이 총 예산보다 작거나 같을 때 카운팅을 한다.(조건이 참일 경우만 +1)
- -- 

## 회고
- 문제를 잘못 이해함.
  - 총 예산과 정확히 일치하는 경우의 수를 찾는 줄 알았지만 총 예산 이상에 해당하는 경우의 수를 찾는 문제였음.
  - 로직에서 조건을 if (sum == budget) 라고 부여했는데 같은 경우가 아닌 총 예산(budget) 이상인 경우를 조건으로 부여해야 함
  - if (sum <= budget)

- 오름차순 정렬 의도 : 총 예산에서 지원한 부서의 개수를 최대한으로 뽑아내기 위함
- 즉, 오름차순 정렬을 우선적으로 시행하여 최대로 계산되는 개수를 증가시키기 위함
