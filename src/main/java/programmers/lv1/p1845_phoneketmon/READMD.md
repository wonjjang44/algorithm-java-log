# p1845 - 폰켓몬

## 요약
자신의 연구실에 있는 총 N 마리의 폰켓몬 중에서 N/2마리를 가져가도 좋다고 함.<br/>
폰켓몬은 종류에 따라 번호를 붙여 구분한다. 따라서 같은 종류의 폰켓몬은 같은 번호를 가지고 있다.<br/>
ex) 연구실에 총 4마리의 폰켓몬이 있고, 각 폰켓몬의 종류 번호가 [3번, 1번, 2번, 3번]이라면<br/>
이는 3번 폰켓몬 두 마리, 1번 폰켓몬 한 마리, 2번 폰켓몬 한 마리가 있음을 나타낸다.<br/>
이때, 4마리의 폰켓몬 중 2마리를 고르는 방법은 다음과 같이 6가지가 있다.
> - 첫 번째(3번), 두 번째(1번) 폰켓몬을 선택
> - 첫 번째(3번), 세 번째(2번) 폰켓몬을 선택
> - 첫 번째(3번), 네 번째(3번) 폰켓몬을 선택
> - 두 번째(1번), 세 번째(2번) 폰켓몬을 선택
> - 두 번째(1번), 네 번째(3번) 폰켓몬을 선택
> - 세 번째(2번), 네 번째(3번) 폰켓몬을 선택

이때, 첫 번째(3번) 폰켓몬과 네 번째(3번) 폰켓몬을 선택하는 방법은 한 종류(3번 폰켓몬 두 마리)의 폰켓몬만 가질 수 있지만,<br/>
다른 방법들은 모두 두 종류의 폰켓몬을 가질 수 있다.<br/>
따라서 위 예시에서 가질 수 있는 폰켓몬 종류 수의 최댓값은 2가 된다.<br/>
최대한 다양한 종류의 폰켓몬을 가지길 원하기 때문에, 최대한 많은 종류의 폰켓몬을 포함해서 N/2마리를 선택하려 함.<br/>
N / 2마리의 폰켓몬을 선택하는 방법 중, 가장 많은 종류의 폰켓몬을 선택할 수 있도록 메서드 구현.

## 접근
- 최대한 다양한 종류의 폰켓몬을 가지길 원함 => 중복 제거

## 회고
- 조합의 경우의 수를 뽑는 줄 알았어.
- 중요한 것은 조합의 수가 아니라 최대 몇 종류까지 가능한지를 뽑아야해.
- [3, 1, 2, 3] => 여기서 종류의 개수는 몇일까? => 3, Y? => 유일한 값만 세야 하니까
  - 중복 제거가 필요해.
  - Set 자료구조를 사용해야겠다고 생각했어.
- Set의 size를 리턴하도록 로직을 작성했고, 그 결과 첫 번째 테스트에서 걸렸어.
- N / 2를 고려하지 않은 것 같아.
- 만약 [1, 2, 3, 4, 5, 6] 이 있을 때, 
  - 전체 : 6마리
  - 고를 수 있는 수 : 3(6 / 2)
  - 종류 : 6종류
- 위 예제에서 난 종류만 리턴한 꼴.
- 종류가 6종류라도 내가 선택할 수 있는 수는 전체에서 절반만 선택할 수 있다고 했어.
- 즉, 6종류 중 단 3마리만 선택할 수 있는거지
- 그렇기 때문에 최대 종류는 3


- 다른 예시는?
  - [1, 1, 1, 3, 3, 3]
    - 전체 6마리
    - 고를 수 있는 수 : 3(6 / 2)
    - 종류 : 2종류
  - 여기선 고를 수 있는 수가 3마리지만 종류는 2종류 밖에 없어.
  - 종류가 2종류 뿐이라서 (1, 1), (1, 3), .. 어떻게 해도 2종류 조합만 만들어져.
  - 그렇기에 최대 종류는 2가 돼.

즉, 최대 종류는 종류 수의 최솟값이야. => min(종류 수, 고를 수 있는 수)
