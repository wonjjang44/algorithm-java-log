# p12930 - 이상한 문자 만들기

- 링크: https://school.programmers.co.kr/learn/courses/30/lessons/12930
- --

## 요약
문자열 s에 대하여 각 단어는 하나 이상의 공백으로 구분돼 있다.<br/>
각 단어의 짝수번째 알파벳은 대문자로, 홀수번째 알파벳은 소문자로 변경하여 리턴하는 메서드 구현.
- --

## 접근
- split을 사용하여 공백 기준으로 배열로 생성한다
- 해당 배열에서 루프를 돌면서 0번째 단어(ex_try)에서 짝수번째 인덱스를 분류하여 대문자로 변경한다.
- -- 

## 회고
- 위 접근 방식대로 구현은 했지만 띄워쓰기 추가가 문제가 됨
  - 내부 반복이 끝날 때마다 append(" ");를 추가함.
  - 맨 마지막에 공백이 붙음.
  - substring으로 잘랐더니 테스트에서 통과하지 못함(무조건 뒷 공백을 자르고 있으므로)
    - 반례) "  TRy HElLo  WORLD " => "  TrY HeLlO  WoRlD "


- split으로 자르고 배열로 만드는 방법은 잘못됨.
  - 특히 split 메서드는 마지막 공백을 무시함 => [, , TRy, HElLo, , WORLD]
- 원본 문자열을 그대로 순회하되, 공백을 만나면 그대로 공백을 추가한다.
  - i로 짝수를 판단하면 공백이 있는 자리에 그대로 또 공백을 추가해버리고 있음.
  - 즉 단어별로 짝수를 판별해야 하는데 글자 하나씩에 대하여 판별하고 있었음.(문제 조건과 상이해짐)
  - 이를 구별할 수 있는 다른 변수가 필요할 듯(하나의 단어가 종료됐다 라는 것을 알 수 있는?)


- split을 사용하는 로직을 사용하려면 split(" ", -1); 두 번째 인자값으로 -1도 넘겨주면 문자열 뒤 공백도 같이 출력됨.
- 문제는 공백 추가인데 문자열 뒷부분에 공백이 항상 추가되고 있음.
- 공백 기준 마지막 단어 전까지만 공백을 붙이는 로직 추가
