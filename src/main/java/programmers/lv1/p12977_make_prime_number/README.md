## 회고
- 우선적으로 3가지 조합을 어떻게 뽑아낼지가 제일 고민이였어
- 1, 2, 3, 4 기준으로 노트에는 아래와 같이 뽑아내긴 했어.(대괄호 안의 값은 인덱스 값)
  > - 1, 2, 3 => [0, 1, 2]
  > - 1, 2, 4 => [0, 1, 3]
  > - 1, 3, 4 => [0, 2, 3]
  > - 2, 3, 4 => [1, 2, 3]

  - 총 4가지 경우가 나옴
  - 각 인덱스 자리를 아래와 같이 치환하면?
  - [i, j, k]
    - i는 0과 1
      - i가 0일 때, j는 1 또는 2
        - j가 1일 때, k는 2 또는 3
          - k가 2일 때
            - => [0, 1, 2]
          - k가 3일 때
            - => [0, 1, 3]
            
        - j가 2일 때, k는 3
          - k가 3일 때
            - => [0, 2, 3]

      - i가 2일 때
        - j는 2
          - j가 2일 때, k는 3
            - => [1, 2, 3]
            
    - 총 4가지 경우
    - 이 때, i < j < k

- 만약 4가지 조합으로 뽑아낸다면? => 4중 for문 i= 0 ~ i < length - 3, j = i + 1 ~ j < length - 2, k = j + 1 ~ k < length -1, m = k + 1 ~ m < length   

- 조합 로직을 이번에 잡았어.
- 이 문제에 다른 핵심 포인트는 소수 걸러내기야.
- 난 기존에 2부터 n - 1까지 하나씩 루프를 돌면서 % 연산 결과가 0인 것을 찾아냈어.
- 그런데 이 방법은 O(N) 시간복잡도를 가져.
- 그렇기에 제곱근을 활용한 방식으로 변경했어.
- 큰 약수는 항상 작은 약수와 짝을 이룬다 라는 것을 이용하여 연산 횟수를 절반으로 줄일 수 있어
