# p12906 - 같은 숫자는 싫어

- 링크: https://school.programmers.co.kr/learn/courses/30/lessons/12906
- --

## 요약
0~9까지의 정수 배열이 주어질 때, 연속적으로 나타나는 숫자는 하나만 남기고 나머지는 전부 제거한 배열을 리턴.<br/>
리턴 시 정수 배열의 순서를 유지해야 한다.<br/>
> - arr = [1, 1, 3, 3, 0, 1, 1] 이면 [1, 3, 0, 1] 을 리턴.
> - arr = [4, 4, 4, 3, 3] 이면 [4, 3] 을 리턴.
- --

## 접근
- 첫 번째 값을 기준으로 루프를 돌면서 같은지 비교한다.
  - 최초 비교 기준값을 먼저 담아 놓음.
- 값이 다를 경우만 기준을 변경해주고 answer 배열에 담는다.
- 루프는 1부터 시작.(이미 기준값을 담아 놓았으므로)
- answer 배열의 크기를 arr의 length로 잡아 놓았기 때문에 0이 들어감.
  - Arrays.copyOf() 메서드를 사용하여 인덱스 0번부터 idx만큼 복사한다.
- -- 

## 회고
- 이중 for문을 돌면서 불필요한 비교 연산을 진행하여 값이 꼬임.
  - 연속된 값, 즉 현재값과 직전값만 비교하면 되는데 배열 길이만큼 끝까지 계산하고 있었음.
