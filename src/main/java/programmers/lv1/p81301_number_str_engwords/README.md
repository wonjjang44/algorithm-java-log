# p81301 - 숫자 문자열과 영단어

- 링크: https://school.programmers.co.kr/learn/courses/30/lessons/81301
- --

## 요약
영단어로 이루어진 문자열을 숫자 자릿수로 변경하여 리턴하는 메서드 구현.<br/>
ex) one4seveneight => 1478 / 23four5six7 => 234567 <br/>
문자열에 숫자가 기입돼 있다면 숫자는 건너뛴다.
- --

## 접근
- 루프를 돌면서 문자열에 숫자가 포함돼 있다면 패스하고 문자열만 비교한다.
- 최초 0~9까지의 자릿수를 영어로 표기한 배열을 선언한다.(ex_one, two, ...)
- 숫자를 제외한 문자열과 영어로 표기된 자릿수 배열을 비교하여 치환한다.
- -- 

## 회고
- charAt 메서드로 뽑아낸 문자열 각 한 자리마다 byte 타입으로 맞추고 아스키 코드 배열과 비교함.
- 숫자 아스키 코드 값이 아스키 배열에 존재한다면 그 값은 제외.
  - 아니라면 정규식 사용?
- 위 방식대로 적용하려고 했지만 실패하여 걸국 힌트를 얻음.
  - 0~9 아스키 코드를 배열로 선언 후 이 값과 비교((byte) c != asciiArr[j])하는 방식은 버림
  - '0' <= c && c <= '9' 방식을 사용. '0' => 48 / '9' => 57
    - char 끼리 비교 시 숫자 비교가 됨('0' => 내부적으로 정수값)
- 문자열과 숫자를 분리하는데 성공했지만 영단어를 만났을 때, 어디까지 읽어야 하는 기준을 못잡음...

- '0' <= c && c <= '9' 조건(숫자일 경우)이 true 일 경우 변환할 필요x => 그대로 append
- false라면 반복하면서 한 글자씩 engWord에 append함.(ex_"o" + "n" + "e")
- 곧바로 루프를 돌면서 한 글자씩 append한 문자열이 wordsArr 배열의 값과 같은지 비교
  - 같다면 engWord 초기화 후 최종적으로 리턴할 숫자 문자열(numStr)에 append 해준다.

- 힌트를 얻었으니 다시 한 번 더 풀어봐야 함.
